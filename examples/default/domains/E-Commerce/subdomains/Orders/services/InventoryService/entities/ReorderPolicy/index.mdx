## Overview

The **Reorder Policy** is a value object that encapsulates the restocking strategy for a stock item. It defines the thresholds and quantities used by the inventory service to decide when to raise reorder events.

## Reorder Logic

```
if quantityAvailable <= reorderPoint AND isAutoReorderEnabled:
    reorderAmount = min(reorderQuantity, maximumStockLevel - quantityOnHand)
    raise ReorderRequested event
```

## Business Rules

* `reorderPoint` must be greater than or equal to `safetyStockLevel`
* `maximumStockLevel` must be greater than `reorderPoint + reorderQuantity`
* `leadTimeDays` must be positive
* When `isAutoReorderEnabled` is false, only alerts are raised (no automatic purchase orders)
