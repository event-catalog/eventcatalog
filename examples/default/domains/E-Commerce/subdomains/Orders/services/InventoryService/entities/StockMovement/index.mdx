## Overview

A **Stock Movement** is an immutable event-sourced record capturing every change to a stock item's quantity. It serves as the complete audit trail for the inventory bounded context.

Movements are append-only — they are never updated or deleted. Corrections are recorded as new movements of type `ADJUSTMENT`.

## Movement Types

| Type | Description |
|------|-------------|
| INBOUND | New stock received from a supplier |
| OUTBOUND | Stock shipped to fulfill an order |
| TRANSFER | Stock moved between warehouses |
| ADJUSTMENT | Manual correction (damaged, miscounted, etc.) |
| RETURN | Stock returned by a customer |

## Relationships

* **StockItem:** Every movement belongs to exactly one stock item
* **Warehouse:** Transfers reference both a source and destination warehouse
* **Reservation:** Outbound movements may link to the reservation they fulfilled

## Business Rules

* Movements are immutable once recorded
* `TRANSFER` movements must have both `sourceWarehouseId` and `destinationWarehouseId`
* `OUTBOUND` movements must have a negative quantity
* `INBOUND` and `RETURN` movements must have a positive quantity
* Each movement triggers an `InventoryAdjusted` domain event
