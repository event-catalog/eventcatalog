## Overview

The **Stock Item** is the aggregate root of the Inventory bounded context. It encapsulates all invariants related to stock levels, reservations, and reorder decisions for a single SKU within a specific warehouse.

All state changes — reservations, adjustments, transfers — are coordinated through this aggregate to maintain consistency.

## Aggregate Invariants

* `quantityAvailable` must always equal `quantityOnHand - quantityReserved`
* `quantityOnHand` cannot be negative
* `quantityReserved` cannot exceed `quantityOnHand`
* A reservation can only be created if `quantityAvailable >= requested amount`
* When `quantityAvailable <= reorderPolicy.reorderPoint`, an `OutOfStock` or reorder event is raised

## Lifecycle

1. **Created** when a new SKU is introduced to a warehouse
2. **Reserved** when an order claims units (creates a `Reservation`)
3. **Adjusted** when physical stock changes (creates a `StockMovement`)
4. **Discontinued** when the product is removed from catalog
5. **Quarantined** when stock fails quality checks

## Relationships

* **Warehouse:** Each stock item belongs to exactly one `Warehouse`
* **ReorderPolicy:** Governs automated restocking thresholds and quantities
* **Reservation:** Temporary holds placed by orders before fulfillment
* **StockMovement:** Immutable audit log of every quantity change
