EventCatalog Modeling helps you define architecture as code in `.ec` files, review changes in Git, and sync those models with EventCatalog.

:::caution[Beta]
The `.ec` docs experience is in beta.
Share feedback in [GitHub Issues](https://github.com/event-catalog/eventcatalog/issues/new).
:::

## What `.ec` is

- A text-based modeling format for event-driven architecture resources.
- Supports services, messages, domains, data products, containers, teams, and users.
- Built for fast iteration with Git diffs and pull request review.

## Workflow at a glance

1. Model resources in `.ec` files.
2. Review and approve changes in Git.
3. Import models into EventCatalog.
4. Export back to `.ec` as you iterate.
